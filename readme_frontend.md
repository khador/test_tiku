# 前端开发需求单

## 一、 全局架构与安全规范 (重点)

* **HTTP 状态码与路由守卫规范：**
  * **401 Unauthorized (登录失效)**：拦截器捕获后，必须清除本地存储的 Token，强制重定向至 `/login` 页面。
  * **403 Forbidden (越权访问)**：拦截器捕获后，**不要清除 Token**。应弹出 Message 提示“无权限访问该页面”，并根据本地 Token 的 `role` 自动回退至对应的 Dashboard。
* **设备适配：**
  * **学生端 (Pad)**：采用触控友好设计（如 Vant / 自定义响应式卡片），适配横竖屏，输入框需防止软键盘遮挡。
  * **教师/管理员端 (PC)**：采用 Element-Plus / Ant Design Vue，侧边栏布局，重度依赖宽屏数据表格。
* **富文本防线：**
  * 渲染题干前，必须将 HTML 字符串中的图片 `<img src="/images/...">` 正则替换为 `http://[后端域名]/images/...`。
  * 建议使用 `DOMPurify` 处理后端 HTML，并设置局部 CSS `img { max-width: 100%; }` 防止排版错乱。

## 二、 管理员工作台 (PC 端)
* **账号管理 (CRUD)**: 
  - 教师列表：支持新增、编辑档案、重置密码、禁用/启用账号。
  - 学生列表：支持按姓名/学号搜索，新增和编辑档案。
* **班级配置**: 
  - 支持创建班级并分配负责教师。
  - (*注：接口已由后端 DRF ModelViewSet 提供，支持标准 RESTful 操作*)

## 三、 教师端中心 (PC 端)
* **学情看板 (`/teacher/dashboard/`)**: 
  - 展示名下班级人数。
  - 表格展示学生练题次数、平均正确率（推荐使用带有颜色区分的进度条组件展示率值）。
* **知识点盲区分析 (`/teacher/questions-analysis/`)**: 
  - 展示错误率 Top 10 错题排行榜。
  - 列表区域展示 `stem_preview` 和 `error_rate`。
  - **交互要求**：点击某道题时，弹出 Dialog 或 Drawer，**直接使用当前接口返回数据的 `stem_full`、`answer` 和 `analysis` 字段**进行详情渲染（后端已做好数据冗余，无需二次请求单独的详情接口）。

## 四、 学生端中心 (Pad 端)
* **沉浸式刷题 (`/practices/generate/` & `submit/`)**: 
  - 顶部常驻正向计时器。
  - 交卷后立刻切换至“红绿模式”（做对标绿，做错标红），并在错题下方展开 `analysis`。
* **错题消除计划 (`/practices/error_book/`)**: 
  - 列表展示错题时，利用后端返回的 `consecutive_correct` 字段，以醒目的 UI（例如两颗星⭐️）提示学生“再连续做对 X 次即可消灭该错题”。
* **训练足迹 (`/practices/history/`)**: 
  - 放弃复杂 Table，采用 Pad 友好的大卡片（Card）流展示历次练习用时与成绩。
